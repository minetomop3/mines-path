# 個人WEBサイト・ブログ開発仕様書

## 1. プロジェクト概要

* **サイト名（表示名）**: `mines-path 〜豊かさを創る、自分と道〜`
* **レポジトリ名**: `mines-path`
* **コンセプト**: 石嶺氏（Mine）の「私自身（Mine）」と歩む「道（Path）」であり、「宝の山（Mine）＝豊かさ」を創り出す場所。日々の学びが豊かさにつながっていく軌跡を表現する。

ユーザー個人の情報発信、および創作物を公開するためのWEBサイトを構築する。
現在Noteで展開している約50件の多様なコンテンツ（MBA経営学、プログラミング、エッセイ、小説など）を**AIによって分析・再整理（カテゴリ最適化・SEO対策）**し、収益化に繋がりやすい分かりやすい構造に再構築することを最大の目的とする。

## 2. 開発フェーズ Phase 1: コンテンツのAI分析と最適化（最重要ステップ）

サイトのUIを作る前に、バラバラになっている50件のNote記事を整理し、収益化の基盤を作ります。Claude Codeに以下のタスクを指示します。

1. **データ読み込み**: ユーザーが `note_exports` フォルダに入れたNoteのエクスポートデータをすべて読み込む。
2. **AIによる分類とタグ付け**: AIが各記事の内容を読み取り、収益化（特定のテーマに特化した広告表示など）に直結しやすい、以下のような整理されたカテゴリ・タグに自動で再分類する。
   * 例: `[経営学/MBA]`, `[AI・プログラミング]`, `[日々のエッセイ]`, `[小説・創作]`
3. **SEOと収益化に向けたメタデータ付与**: 全記事に対して、検索エンジンで引っかかりやすくなるような `Title`, `Meta Description`, `Keywords` をAIに自動生成させ、記事データの先頭（Frontmatter）に埋め込む。
4. **Markdownへの出力**: 整理し終わったデータを、美しいMarkdown形式としてプロジェクト内の `src/content/blog/` フォルダに出力する。

## 3. ゴールと主な機能

* **プラットフォーム**: GitHub Pagesを利用した無料・高速な静的サイトホスティング。
* **技術スタック**: 
  * フレームワーク: **Astro** (AIが生成した整理済みのMarkdown形式と最高に相性が良く、SEO爆速のフレームワーク) 
  * スタイリング: TailwindCSS または Vanilla CSS
* **主な機能**:
  * **カテゴリ・タグごとの動的ページ**: AIが整理したカテゴリごとに、自動的に美しい一覧ページが生成される。
  * **動く小説（リッチコンテンツ）機能**: スクロールアニメーション等を組み込んだ特設ページ。
  * **収益化・広告枠の配置**: 整理されたカテゴリに連動させて、Google AdSenseやアフィリエイトリンクを効果的に配置するコンポーネント。

## 4. ページ構成（サイトマップ）

1. **Home（トップページ）**
   * サイトの顔となるファーストビュー
   * **AIが整理したカテゴリ別**の最新記事ピックアップ
2. **Category / Blog（記事一覧ページ）**
   * カテゴリ・タグでの絞り込み機能
3. **Article（ブログ記事詳細ページ）**
   * AIが付与したSEOメタデータを出力（Helmet/Head機能）
   * 広告コンポーネントの配置
4. **Interactive Novels（動く小説 特集一覧） & Viewer**
   * 没入感のある全画面スクロールUI
5. **About（プロフィールページ）**

## 5. デザイン要件 (Design & Aesthetics)

* **知性とクリエイティビティの融合**: 経営学の論理的・知的な落ち着きと、小説やAIの先進性が同居するデザイン。
* **プレミアムな印象**: 余白の美しさ、読みやすいタイポグラフィ（Noto Sans JPや明朝体の使い分け）、ホバー時の微細なアニメーションを活用する。

## 6. AI（Claude Code）への指示: 実装の全体フロー

この仕様書を読み込んだ後、以下の手順で開発を進めてください。

1. **[Phase 1]** ユーザーが `note_exports` に配置したデータを読み込むNode.jsスクリプトを作成し、「記事の要約・分類（カテゴライズ）・SEOメタデータ（Title/Description）生成」をAI API（またはClaude自身の処理能力）を用いて行い、結果を `src/content/blog/` にMarkdownで保存すること。
2. **[Phase 2]** `npx create-astro@latest` でプロジェクトを初期化し、先ほどのMarkdownを高速で読み込むAstroの「Content Collections」を設定すること。
3. **[Phase 3]** UIとデザイン（グローバルCSS、ヘッダー・フッター、カテゴリ別トップページ、記事詳細ページ）を実装すること。記事詳細には広告用のプレースホルダを設けること。
4. **[Phase 4]** 動く小説用のサンプルページとして、スクロール連動のアニメーションを備えたテストページを実装すること。
